# Mini-Projects-OS

## Steps to contribute on this project

```
 1. Fork the project to your own GitHub account.
 2. Clone the forked project to your local development environment.
 3. Create a new branch for your changes.
 4. Make the changes to the codebase and test them locally.
 5. Commit your changes to the new branch.
 6. Make sure that branch you create should be named same as the issue you've chosen.
 7. Push the new branch to your forked GitHub repository.
 8. Create a pull request from your forked repository to the original project.
 9. Wait for the maintainers to review and merge your changes.
 10. Keep your fork synced with the original repository.
 ```
